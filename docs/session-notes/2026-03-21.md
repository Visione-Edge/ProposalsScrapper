## Decisions Made

- **Source client protocol over inheritance**: Used a `SourceClient` protocol (structural typing) rather than an ABC, keeping existing SICOPClient untouched and wrapping it in an adapter. Why: minimizes disruption to proven code.
- **`sources/` is additive, `sicop/` is preserved**: The existing SICOP folder was not refactored into the new sources structure. Why: scanner, storage, classifier, and notifier are tightly coupled to SICOP internals; separating them would be high-risk refactor for no immediate gain.
- **Auto-migration on startup**: Source columns added via `ALTER TABLE` with existence check rather than a migration runner. Why: simple, no migration tooling dependency, safe to run repeatedly.
- **Expiration filter is conservative**: `_is_expired()` only removes tenders with a confirmed past deadline OR 6+ months old with no deadline. Why: avoids false expiration of tenders with ambiguous dates.
- **Rate limit at 60/hour for testing**: Temporarily raised from 2/hour. Must be reverted before production deployment.
- **ICE/PEL and UNGM disabled by default**: Both require capabilities (authenticated sessions, headless browser) not available in the current environment. Stubbed out to document the requirement without breaking the scan loop.
- **World Bank switched to text search**: Initial implementation used faceted search; switched to `qterm` text search for better recall. Why: faceted search returned too few results for Costa Rica procurement context.
